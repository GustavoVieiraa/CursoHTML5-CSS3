Capítulo 02 Aula 02 – Domínio e Hospedagem

Dados Guardado no Servidor!

Os dados do nosso site, por exemplo estão armazenados dentro do nosso servidor. Podemos imaginar, que temos diversos Pontos (A, B, C, D) que são computadores realizando solicitações ao servidor (S). Mas, importante entender que, o Site, toda nossa parte de desenvolvimento HTML e CSS só irão funcionar se estiverem no servidor.

Nossa URL

Então, podemos pensar que nossa URL é a ponte do nosso computador (Cliente) para o Servidor. E para ter esse acesso, a primeira coisa que precisamos ter é o Domínio. E para guardar os nossos arquivos HTML e CSS, nós utilizamos o termo como HOSPEDAGEM. Geralmente, hospedamos nossos sites e projetos em servidores pagos com serviços pre-moldados, como HostGator,

DOMÍNIO

-> o Domínio é um nome ÚNICO!
-> Pago Anualmente! (Existem pacotes de 1, 3, 5, 10 anos).
-> Vários TLDs

HOSPEDAGEM

-> Espaço para armazenar arquivos.
-> Pago Mensalmente.
-> A escolha da hospedagem é por: Espaço, Memória, recursos.

O que é URL?

Exemplo de URL: www.github.com/gustavovieira
A URL ela é dividida em partes.
Em nosso exemplo: github -> é o nosso domínio.
.com -> é o nosso TLD.
www -> é o sub-domínio.
/gustavovieira -> é o nosso Caminho dentro do URL.
